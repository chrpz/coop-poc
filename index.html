🔓 Missing Cross-Origin-Opener-Policy (COOP) Header

⚠️ Security Advisory — This finding has been identified on your web application and should be remediated at your earliest convenience.

What is the Risk?

Your application is missing the Cross-Origin-Opener-Policy (COOP) HTTP response header. Without it, an external malicious website that tricks a user into clicking a link can open your application in a new browser window — and silently navigate it to a phishing page while the user believes they are still on your legitimate site.

This can lead to credential theft, session hijacking, or user impersonation without any visible warning to the victim.

Impact

  • Users can be silently redirected away from your application
  • Sensitive information may be harvested via a lookalike phishing page
  • No user interaction beyond the initial click is required

Recommended Fix

✅ Add the following HTTP response header to your web server or application:

Cross-Origin-Opener-Policy: same-origin

This severs the browser's opener reference between windows, preventing any external page from retaining control over your application's window after it has been opened.
